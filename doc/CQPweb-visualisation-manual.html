  1. Introduction to visualisations


In CQPweb, the term visualization refers generically to any aspect of how the concordances produced by the CQP back-end are translated into the HTML that you see in your browser in CQPweb.


Visualisation has been made as customisable as possible. CQP allows any p-attributes or s-attributes to be displayed in the concordance.   1. XML visualisations


In the same configuration page, you can install, review and update/delete XML visualisations for the same


XML visualisations must be input using a specialised variant of the “BBCode” simplified pseudo-HTML markup. You can read about BBCode here.


Why not just use HTML to specify the visualisations?


Because CQPweb would then be vulnerable to certain nasty attacks. HTML can safely be written directly into code files (such as the configuration file) but not via the web. So, it was a choice between letting the visualisation be specified over the web (with limitations on what you can do), or forcing all visualisations to be entered manually into code files. We've adopted the first route.

You can also use embedded variables to represent the values of different fields of relevance to the XML tag that is being rendered. These are specified as follows:


$$$attribute-name$$$


renders the value of <tag attribute-name="..." > in the original corpus.


$$$$$$


renders the single value of the current s-attribute instance, if it has one (depends on how the corpus was indexed....)


~~~field-name~~~


renders the value of text-metadata field field-name for the text in which the current s-attribute instance occurs.


~~~~~~


renders the text ID code of the current text.


Note that the same value can be referenced at different levels. For instance, assume we have insdexed <sentence n="1">...</sentence>, <sentence n="2">...</sentence> and so on. In a visualisation for sentence, we would reference the sentence number as $$$n$$$. But as sentence_n is itself an s-attribute, we could create a visualisation for that, where the sentence number would be referred to as $$$$$$.
